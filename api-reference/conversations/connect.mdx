---
title: "Connect Conversation"
openapi: "openapi/akapulu.json POST /conversations/connect"
---

Use this endpoint to create a conversation session and receive:

- `room_url`
- `token`
- `conversation_session_id`

Required headers:

- `Authorization: Bearer <YOUR_AKAPULU_API_KEY>`
- `Content-Type: application/json`

Required request fields:

- `scenario_id` for the scenario to run
- `avatar_id` for the avatar to use in this conversation

Optional request fields include:

- `runtime_vars` for template/runtime values
- `stt_keywords` are optional keywords used in the speech-to-text step of the pipeline to improve recognition of specific words in user speech (maximum 5)
- `voice_only_mode` for audio-only calls
- `record_conversation` whether to record this conversation session (default `false`)
- `wait_for_client_ready` controls when the session is marked ready to begin (default `true`)
  - when `true`, Akapulu waits until your embedded client has fully connected before marking the call ready and starting the conversation flow
  - when `false`, Akapulu skips that client-ready wait and proceeds once call participants are ready
  - use `true` for embedded SDK/client integrations (recommended)
  - use `false` for Daily prebuilt UI

Example request body:

```json
{
  "scenario_id": "scenario_1234",
  "avatar_id": "avatar_5678",
  "runtime_vars": {
    "today": "2026-03-19"
  },
  "stt_keywords": ["Akapulu", "Dr. Nguyen", "A1C"],
  "voice_only_mode": false,
  "record_conversation": true,
  "wait_for_client_ready": true
}
```
