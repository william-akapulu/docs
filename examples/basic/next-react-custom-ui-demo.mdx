Use this example when you want full control over the conversation experience while still building on the Akapulu Web SDK.

## What this example shows

- using `@akapulu/react` without `@akapulu/react-ui`
- building a fully custom call surface with your own layout
- rendering video, transcript, node state, tool events, and media controls manually
- using `@akapulu/server` in local Next.js API routes
- redirecting to a conversation details page after the call ends

## Routes in this example

- `/` renders the full custom UI
- `/view-conversation-details/[conversationId]` shows post-call detail review

Local API handlers:

- `app/api/akapulu/connect/route.ts`
- `app/api/akapulu/updates/route.ts`
- `app/api/conversation-details/route.ts`
- `app/api/view-recording/route.ts`

## Quickstart

1. Install dependencies with `npm install`.
2. Copy `.env.example` to `.env.local`.
3. Set `AKAPULU_API_KEY`.
4. Update `SCENARIO_ID` in `app/api/akapulu/connect/route.ts`.
5. Run `npm run dev`.
6. Open `http://localhost:3000`.
