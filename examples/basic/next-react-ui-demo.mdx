Use this example when you want to ship quickly with Akapulu's prebuilt UI components.

## What this example shows

- using `@akapulu/react-ui` for the conversation interface
- using `@akapulu/server` in local Next.js API routes
- a baseline prebuilt UI at `/default`
- a customized prebuilt UI at `/customized`
- conversation detail and recording review after a call ends

## Routes in this example

- `/default` uses the baseline `AkapuluConversation` flow
- `/customized` uses the same prebuilt UI with styling and behavior overrides

Both routes use local API handlers that call Akapulu from the server:

- `app/api/default/akapulu/connect/route.ts`
- `app/api/default/akapulu/updates/route.ts`
- `app/api/customized/akapulu/connect/route.ts`
- `app/api/customized/akapulu/updates/route.ts`
- `app/api/conversation-details/route.ts`
- `app/api/view-recording/route.ts`

## Quickstart

1. Install dependencies with `npm install`.
2. Copy `.env.example` to `.env.local`.
3. Set `AKAPULU_API_KEY`.
4. Update `SCENARIO_ID` in the connect route you want to test.
5. Run `npm run dev`.
6. Open `http://localhost:3000/default` or `http://localhost:3000/customized`.
