---
title: "Context Management"
description: "How to keep conversational context clear, focused, and reliable across scenario stages."
---

Context management is how you decide what the assistant should carry forward as a conversation progresses.

In Akapulu, this usually means:

- keeping a clear global role/persona throughout the full call
- giving each stage only the context it needs for that moment
- using tool outputs and runtime variables to ground responses
- transitioning stages when enough information has been collected

Good context management helps the assistant stay consistent, avoid repeated questions, and move the conversation forward predictably.

For foundational setup, start with [Scenarios Overview](/guides/scenarios/overview) and [Node Basics](/guides/scenarios/node-basics).
